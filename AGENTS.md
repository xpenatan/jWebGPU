# AGENTS.md - jWebGPU

Token-light operational guide for coding agents in this repository.

## 1) Start Here (Mandatory)

Must do in this order:

1. Read `CURRENT_CHAT_CONTEXT.MD` first.
2. Validate the request against real source/APIs/tasks (no guessing).
3. Record plan + target files in `CURRENT_CHAT_CONTEXT.MD`.
4. Get user approval before execution.
5. Execute edits/commands.
6. Update `CURRENT_CHAT_CONTEXT.MD` to current state only.
7. Run at least one compile/verification task before ending turn.

`CURRENT_CHAT_CONTEXT.MD` is rolling state, not a running log: replace/update prior entries instead of appending history.

## 2) Project Snapshot

- jWebGPU is a generated Java WebGPU binding stack for desktop, Android, and web.
- Pipeline shape: `webgpu-base` (hand-authored templates) + `webgpu-build` (generator/native build driver) -> generated and packaged target modules.
- Generation entrypoint is `webgpu/webgpu-build/src/main/java/WGPUBuild.java`.
- Java API shape comes from `webgpu-base` + `webgpu-build/src/main/cpp/jWebGPU.idl`; custom native glue is in `webgpu-build/src/main/cpp/custom/jWebGPU.cpp/.h`.

## 3) Source Of Truth (Edit vs Generated)

- Edit source modules:
  - `webgpu/webgpu-base/src/main/java/**`
  - `webgpu/webgpu-build/src/main/cpp/**`
- Do not hand-edit generated Java in:
  - `webgpu/webgpu-core/src/main/java/**`
  - `webgpu/webgpu-ffm/src/main/java/**`
  - `webgpu/webgpu-web/src/main/java/**`
- Generated files usually include `This file was generated by JParser`.

## 5) Native Block + IDL Conventions

- Native bridge directives live in `webgpu-base` Java comments (for example `/*[-JNI;-NATIVE]*/`, `/*[-FFM;-NATIVE]*/`, `/*[-TEAVM;-REPLACE]*/`).
- IDL binding metadata lives in `webgpu/webgpu-build/src/main/cpp/jWebGPU.idl` (for example `[Ref]`, `[BindTo]`, `[-NEW_OBJECT]`, `[-SKIP]`).
- If generated output looks wrong, inspect in order: `jWebGPU.idl` -> `webgpu-base` directive blocks -> generated target file.

## 7) CI Build Matrix Reference

From `.github/workflows/build_and_upload.yml`:

- Windows: JNI + FFM (`wgpu` and `dawn` on x86_64).
- Linux: JNI + FFM (`wgpu` on x86_64).
- macOS: JNI + FFM (`wgpu` on x86_64 and arm64).
- TeaVM: web Dawn build (`webgpu_build_project_web_dawn`) with emscripten.
- Android: JNI build (`webgpu_build_project_android_wgpu_jni`) with NDK.

## 8) Integration Points

- Local development against jParser is enabled via `includeBuild(...)` in `settings.gradle.kts`.
- Optional local TeaVM substitution exists as commented `includeBuild(...)` entries in `settings.gradle.kts`.
- Publishing/versioning flow is controlled by `gradle.properties` + `buildSrc/src/main/kotlin` scripts.
