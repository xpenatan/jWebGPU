# AGENTS.md - jWebGPU

Token-light operational guide for coding agents in this repository.

## 1) Start Here (Mandatory)

Must do in this order:

1. Read `CURRENT_CHAT_CONTEXT.MD` first.
2. Validate the request against real source/APIs/tasks.
3. Record plan + target files in `CURRENT_CHAT_CONTEXT.MD`.
4. Get user approval before execution.
5. Execute edits/commands.
6. Update `CURRENT_CHAT_CONTEXT.MD` to current state only.
7. Run at least one compile/verification task before ending the turn.

`CURRENT_CHAT_CONTEXT.MD` is rolling state, not a running log: replace/update prior entries instead of appending history.

## 2) Project Snapshot

- jWebGPU is a generated Java WebGPU binding stack for desktop, Android, and web.
- Pipeline shape: `base` hand-authored templates + `builder` IDL/native source + jParser Gradle plugin -> generated and packaged target modules.
- Java API shape comes from `webgpu/base/src/main/java/**` plus `webgpu/builder/src/main/cpp/jWebGPU.idl`.
- Custom native glue lives in `webgpu/builder/src/main/cpp/custom/jWebGPU.cpp/.h`.

## 3) Source Of Truth

Edit source modules:

- `webgpu/base/src/main/java/**`
- `webgpu/builder/src/main/cpp/**`
- Gradle build logic in the owning module build files

Do not hand-edit generated Java in:

- `webgpu/core/src/main/java/**`
- `webgpu/shared/jni/src/main/java/**`
- `webgpu/desktop/ffm/src/main/java/**`
- `webgpu/web/wasm/src/main/java/**`

Generated files usually include `This file was generated by JParser`.

## 4) Module Naming

Library modules:

- `:webgpu:download` - native dependency download tasks
- `:webgpu:builder` - jParser Gradle plugin build driver
- `:webgpu:base` - hand-authored Java templates and directive blocks
- `:webgpu:core` - generated Java API layer
- `:webgpu:shared:jni` - generated JNI Java runtime
- `:webgpu:desktop:jni` - desktop JNI native packaging
- `:webgpu:desktop:ffm` - generated FFM Java runtime and desktop native packaging
- `:webgpu:web:wasm` - TeaVM/WebAssembly runtime packaging
- `:webgpu:android:jni` - Android JNI runtime packaging

Demo app modules:

- `:demos:app:desktop-jni`
- `:demos:app:desktop-ffm`
- `:demos:app:web`
- `:demos:app:android`

## 5) Native Block + IDL Conventions

- Native bridge directives live in `webgpu/base` Java comments, for example `/*[-JNI;-NATIVE]*/`, `/*[-FFM;-NATIVE]*/`, and `/*[-TEAVM;-REPLACE]*/`.
- IDL binding metadata lives in `webgpu/builder/src/main/cpp/jWebGPU.idl`, for example `[Ref]`, `[BindTo]`, `[-NEW_OBJECT]`, and `[-SKIP]`.
- If generated output looks wrong, inspect in order: `jWebGPU.idl` -> `webgpu/base` directive blocks -> generated target file.

## 6) Build/Run Quick Path

Use task names from Gradle files. Download tasks are manual prerequisites for native builds; `jParser_build_*` tasks do not run them automatically.

```powershell
.\gradlew.bat :webgpu:download:webgpu_download_glfw_windows
.\gradlew.bat :webgpu:download:webgpu_download_windows_x86_64_wgpu
.\gradlew.bat :webgpu:download:webgpu_download_windows_x86_64_dawn
.\gradlew.bat :webgpu:download:webgpu_download_linux_x86_64_dawn
.\gradlew.bat :webgpu:download:webgpu_download_macos_x86_64_dawn
.\gradlew.bat :webgpu:download:webgpu_download_macos_aarch64_dawn
.\gradlew.bat :webgpu:download:webgpu_download_emdawnwebgpu

.\gradlew.bat :webgpu:builder:jParser_generate
.\gradlew.bat :webgpu:builder:jParser_build_windows64_jni_wgpu
.\gradlew.bat :webgpu:builder:jParser_build_windows64_jni_dawn
.\gradlew.bat :webgpu:builder:jParser_build_windows64_ffm_wgpu
.\gradlew.bat :webgpu:builder:jParser_build_windows64_ffm_dawn
.\gradlew.bat :webgpu:builder:jParser_build_linux64_jni_dawn
.\gradlew.bat :webgpu:builder:jParser_build_linux64_ffm_dawn
.\gradlew.bat :webgpu:builder:jParser_build_mac64_jni_dawn
.\gradlew.bat :webgpu:builder:jParser_build_mac64_ffm_dawn
.\gradlew.bat :webgpu:builder:jParser_build_macArm_jni_dawn
.\gradlew.bat :webgpu:builder:jParser_build_macArm_ffm_dawn
.\gradlew.bat :webgpu:builder:jParser_build_web_wasm
.\gradlew.bat :webgpu:builder:jParser_build_android_jni

.\gradlew.bat :demos:app:desktop-jni:webgpu_demo_app_run_desktop_jni_wgpu
.\gradlew.bat :demos:app:desktop-jni:webgpu_demo_app_run_desktop_jni_dawn
.\gradlew.bat :demos:app:desktop-ffm:webgpu_demo_app_run_desktop_ffm_wgpu
.\gradlew.bat :demos:app:desktop-ffm:webgpu_demo_app_run_desktop_ffm_dawn
.\gradlew.bat :demos:app:web:webgpu_demo_app_run_web
.\gradlew.bat :demos:app:android:installDebug
```

## 7) Validation Minimum

Run the affected workflow, not only compilation. For visual demos, capture the actual target window/browser/device after it is loaded.

At minimum for build logic changes, run:

```powershell
.\gradlew.bat projects
.\gradlew.bat :webgpu:builder:tasks --group jParser --all
```

For generated binding changes, also run the relevant `:webgpu:builder:jParser_build...` task and the runtime module `jar`/`bundleReleaseAar` task that packages its output.
